# PAT 1001 — A+B Format

https://pintia.cn/problem-sets/994805342720868352/exam/problems/type/7

---

## 题目

计算 `a + b`，输出时**每三位加一个逗号**（从右往左数）。

```
输入：-1000000 9
输出：-999,991
```

---

## 思路

1. 算出 `sum = a + b`
2. 负数先输出 `-`，把 sum 取绝对值
3. `sum == 0` 特判直接输出 `0`
4. 用 `sum % 1000` 从低位往高位每三位拆一组，存入数组
5. **倒序**输出：最高位用 `%d`（不加前导零），后面每组用 `,%03d`（不足三位补零）

---

## C 代码

```c
#include <stdio.h>

int main() {
    int a, b;
    scanf("%d %d", &a, &b);
    int sum = a + b;

    // 处理负数
    if (sum < 0) {
        printf("-");
        sum = -sum;
    }

    // 特判 0
    if (sum == 0) {
        printf("0");
        return 0;
    }

    // 每三位拆一组
    int parts[10], cnt = 0;
    while (sum > 0) {
        parts[cnt++] = sum % 1000;
        sum /= 1000;
    }

    // 倒序输出
    printf("%d", parts[cnt - 1]);              // 最高位不加前导零
    for (int i = cnt - 2; i >= 0; i--) {
        printf(",%03d", parts[i]);             // 后面每组补零到 3 位
    }

    return 0;
}
```

---

## 手算示例

```
输入：1234567 + 9 = 1234576

while 循环：
  sum = 1234576
  parts[0] = 1234576 % 1000 = 576,  sum = 1234
  parts[1] = 1234 % 1000    = 234,  sum = 1
  parts[2] = 1 % 1000       = 1,    sum = 0

parts = [576, 234, 1], cnt = 3

输出：
  cnt-1=2: printf("%d", 1)      → "1"
  cnt-2=1: printf(",%03d", 234) → ",234"
  cnt-3=0: printf(",%03d", 576) → ",576"

结果：1,234,576 ✓
```

---

## 不足三位补零示例

```
sum = 1000002

parts[0] = 2,     sum = 1000
parts[1] = 0,     sum = 1
parts[2] = 1,     sum = 0

输出：
  parts[2] = 1      → "1"
  parts[1] = 0      → ",000"   ← %03d 把 0 补成 000
  parts[0] = 2      → ",002"   ← %03d 把 2 补成 002

结果：1,000,002 ✓
```

---

## 关键点

| 细节 | 说明 |
|------|------|
| `%03d` | 不足 3 位自动补 0 |
| 最高位不用 `%03d` | 否则 1000 → `001,000` 而非 `1,000` |
| `sum == 0` 特判 | while 循环对 0 直接跳过，什么都不输出 |
| 负数先打 `-` | 后面逻辑不变，sum 取绝对值 |
