# 最大子数组和 · Kadane 算法讨论实录

> LeetCode 53 / 408 可能考的选择题或代码题  
> 核心：`cur > 0 ? cur + nums[i] : nums[i]`

---

## 一、题目

给你一个整数数组 `nums`，找出一个具有最大和的连续子数组，返回其最大和。

```
输入: [-2,1,-3,4,-1,2,1,-5,4]
输出: 6（子数组 [4,-1,2,1]）
```

---

## 二、算法思想

从左到右遍历，每一步只做一次判断：

**前面累积的和到底是"助力"还是"拖累"？**

```
cur > 0  →  接上（cur = cur + nums[i]）
cur ≤ 0  →  重启（cur = nums[i]）
```

一句话概括：**把中心放在当前数据上，判断它有没有被前面的和拖累。** 前面是正数就接（利润），前面是负数就扔（负债）。

---

## 三、标准代码

```c
int maxSubArray(int* nums, int numsSize) {
    int cur = 0, max = nums[0];
    for (int i = 0; i < numsSize; i++) {
        if (cur > 0)
            cur += nums[i];   // 接上
        else
            cur = nums[i];    // 重启
        if (cur > max) max = cur;
    }
    return max;
}
```

## 四、执行过程演示

`nums = [5, 4, -1, 7, 8]`

```
i=0: cur=5,                max=5
i=1: cur=5+4=9,            max=9
i=2: cur>0 → 接上, 9-1=8, max=9
i=3: cur=8+7=15,           max=15
i=4: cur=15+8=23,          max=23
返回 23
```

`nums = [5, -8, 3]`

```
i=0: cur=5,                 max=5
i=1: cur=5-8=-3,            max=5   ← cur 变成负数了
i=2: cur≤0 → 重启, cur=3,   max=5   ← 不接！重新开始
返回 5
```

---

## 五、关键理解

### Q1：什么时候会触发"重启"？

只有当 `cur` 变成负数时，下一步才重启。

```
cur = 5   → cur + nums[i] 总是 > nums[i]  → 不可能重启
cur = -3  → cur + nums[i] < nums[i]       → 必然重启
```

**只有积累的负数和超过了此前的正数和，cur 才会变成负数。**

### Q2：这道题是局部最优解吗？

不是局部最优。"接上还是重启"的决策看似贪心，但每一步 guard against the worst case：cur 为正时继续接、cur 为负时核爆重来。数学上可以证明这个策略找到的就是全局最大子数组和——这就是 Kadane 算法。

### Q3：这个考点出现在 408 吗？

可能以选择题出现，也可能作为算法设计题的素材。核心就是一个 `if (cur > 0)` 判断，O(n) 时间、O(1) 空间。
